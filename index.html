使い方＆試験の傾向

🎯 案内ページが名指しした「重点」

  • 変数の意味を理解し適切に使えるか（型・整数割り算・オーバーフロー）
  • 条件分岐（特に if / if〜else / if〜else if〜else）を正しく追えるか
  • ループ（特に for / while）を正しく手で追えるか

出題形式（案内より）：「与えられたプログラムの期待される出力を正しく求める」「不足・誤りを正しく追加・修正する」。引っかけ問題や細かい知識問題は出さない。だからトレース力＝1行ずつ手で追う力が命。

🪙 開始3秒前にリロードする反射（型で覚える）

  • 入力 scanf("%d", &x);（& を付ける／配列は scanf("%d", &a[i]);）・実数は %lf
  • 整数同士の割り算は切り捨て：1/2 == 0。実数が欲しければ 1.0/2 か (double)a/b
  • for の型 for(i=0; i<n; i++){ ... }／2重ループは内側 j を i+1 から回すと重複ペア
  • 関数は プロトタイプ宣言→main→定義。int f(int a, int b){ return a+b; }
  • 配列を関数に渡すのは int array[]、呼ぶ時は配列名だけ f(a)（& 不要）

3章 変数と式 S1

狙われるのは①整数割り算の切り捨て ②scanf の書式（%d/%lf）と & ③int のオーバーフロー ④インクリメント前置/後置。

⚠️ ここで死ぬ人が多い落とし穴

  • 1 / i は int ÷ int → 切り捨て。i≥2 で 0 になる。実数和なら 1.0 / i。
  • int を読むのは %d、double を読むのは %lf。型と書式が食い違うと壊れる。
  • 後置 x = a++; は「代入してから増やす」／前置 x = ++a; は「増やしてから代入」。
  • 文字は数：'A' は 65。char c='f'-'F'; は 32（大文字↔小文字の差）。
S1_1修正整数割り算・scanf
1 + 1/2 + 1/3 + … + 1/n を求めたいが、バグが2つあって正しく出ない。2つのバグを直せ。さらに (**) の行を、変数 result 1つだけ・複合代入演算子で書き換えよ。
#include <stdio.h>
int main(void){
    int i,n;
    double result=0;
    printf("Input n>>");
    scanf("%lf",&n);              // ← あやしい(1)
    for( i = 1; i <= n; i++ ){
        result = result + ( 1 / i );  // (**) ← あやしい(2)
    }
    printf("result=%f\n",result);
    return 0;
}
✍️ 紙に「直したscanfの行」と「直した(**)の行」を書いてから開く

解答

バグ1： n は int なのに書式が %lf。→ scanf("%d",&n);

バグ2： 1 / i は整数割り算で i≥2 のとき 0。→ 1.0 / i（分子を実数に）。

(**) の書き換え： result += 1.0 / i;

print文デバッグの例：ループ内に printf("i=%d result=%f\n", i, result); を入れれば、i=2 以降 result が全く増えていない＝1/i が 0 になっていると気づける。
理解度:
S1_2説明オーバーフロー
sum を 2 倍し続けると i=31 で sum=-2147483648 という異常値になる。なぜか。また unsigned int＋書式 %u にするとなぜ正常に出るか、扱える最小・最大値の変化とともに説明せよ。
int i; int sum=1;
for(i=1;i<=31;i++){
    sum=2*sum;
    printf("i=%d → sum=%d\n",i,sum);
}

解答

  • int は 32bit の符号付き整数で、範囲は -2^31 〜 2^31-1＝-2147483648 〜 2147483647。
  • sum は倍々で増え、i=30 で 2^30=1073741824。i=31 で 2^31=2147483648 になろうとするが、これは最大値を1超える。符号ビットに繰り上がり、最小値 -2147483648 に一周（オーバーフロー）する。
  • unsigned int は符号ビットが無いぶん範囲が 0 〜 2^32-1＝0 〜 4294967295。最小値は負なし(0)に、最大値は約2倍に広がる。よって 2147483648 をそのまま %u で表示できる。
理解度:
S1_3作成式・整数演算
給料 pay（15万以上・10の倍数）と外食回数 eatout を入力。①税で10%引かれる ②固定費80000引かれる ③外食は1回5000円。自由に使えるお金 rest を求める1行を補え。
✍️ 「rest = …」の1行を紙に書いてから開く

解答（scanf の直後に追加）

rest = pay - pay/10 - 80000 - 5000*eatout;
pay は10の倍数なので pay/10 は割り切れて誤差なし（＝10%）。pay*9/10 でも可。外食は 5000*eatout。
理解度:
S1_4出力予測前置/後置★
出力を予測せよ（前置・後置インクリメント/デクリメント）。
result=10; x1=result++; printf("line1-> x1=%d result=%d\n",x1,result);
result=10; x2=++result; printf("line2-> x2=%d result=%d\n\n",x2,result);
result=10; y1=--result; printf("line3-> y1=%d result=%d\n",y1,result);
result=10; y2=result--; printf("line4-> y2=%d result=%d\n",y2,result);

解答（実行で確認済み）

line1-> x1=10 result=11 line2-> x2=11 result=11 line3-> y1=9 result=9 line4-> y2=10 result=9
後置 result++／result--：今の値を代入してから 1増減。→ x1=10, y2=10。
前置 ++result／--result：先に1増減してから代入。→ x2=11, y1=9。
どの行も最後の result は 11 or 9（±1 済み）。
理解度:
S1_5〜7発展2進変換・多倍長★★
案内いわく★★は試験水準超え。時間が無ければ後回しでOK。考え方だけ押さえる：2進数→10進は各桁×2の累乗の和、10進→2進は%2と/2の繰り返し、多倍長は各桁を配列に入れて筆算。

参考コード（S1_5 の一例）

int bin[32], i, n, dec=0, w=1;
scanf("%d",&n);                // 例: 1011 を int で受ける
for(i=0; n>0; i++){ bin[i]=n%10; n/=10; }  // 各桁を下位から取り出す
for(int k=0; k<i; k++){ dec += bin[k]*w; w*=2; } // 2^k を掛けて足す
printf("%d\n", dec);
10進→2進(S1_6)は逆に while(n>0){ b[j++]=n%2; n/=2; } で余りを集め、逆順に出す。多倍長(S1_7)は各桁を配列に入れ、下位桁から 繰り上がり を持って足す筆算。
理解度:

4章 制御(1) 条件分岐 S2

最頻出。if〜else if〜else の上から順に評価、&&(かつ)と||(または)の違い、そして「&&/|| の1文」を「入れ子の if」に書き換える変換が繰り返し出る。

⚠️ 条件分岐の落とし穴

  • if〜else if は上から順に調べ、最初に真になった1つだけ実行して残りは飛ばす。
  • A && B＝AかつB（両方真）／A || B＝AまたはB（どちらか真）。取り違えると常に真になったりする。
  • 「範囲内か」は x>=下 && x<=上（&&）。|| にすると全部の値で真になる典型バグ。
  • if(A && B) ⇔ if(A){ if(B){...} }／if(A || B) ⇔ if(A){...} else{ if(B){...} }。
S2_1出力予測if〜else if
入力が 50 / 65 / 80 / 110 のときの各出力を答えよ。
if( score > 100 || score < 0 )   printf(" invalid number \n");
else if( score >= 80 )           printf(" excellent \n");
else if( score >= 70 )           printf(" good \n");
else if( score >= 60 )           printf(" fair \n");
else                             printf(" no good \n");

解答（実行で確認済み）

50 → no good 65 → fair 80 → excellent 110 → invalid number
上から順に判定。65 は 80未満・70未満だが60以上 → fair。80 は最初の80以上で excellent（good以降は飛ばす）。110 は範囲外で invalid。50 はどれも該当せず else の no good。
理解度:
S2_2バグ考察&& と ||
文字を大文字/小文字/数字/その他に判別したいが、正しく動かない。理由を述べ、直せ。（diff = 'f'-'F' = 32、a='a', z='z'）
if(input >= a - diff || input <= z - diff){ printf("英字大文字です\n"); }
else if(input >= a || input <= z ){         printf("英字小文字です\n"); }
else if(input >= '0' || input <= '9'){       printf("数字です\n"); }
else{ printf("英数字以外です\n"); }

解答

理由：「範囲内」は「下限以上 かつ 上限以下」なのに、すべて ||（または）で書いている。|| だと例えば1つ目は「'A'以上 または 'Z'以下」となり、どんな文字でもほぼ必ず真（'Z'以下 or 'A'以上のどちらかは必ず成立）。よって最初の枝にすべて吸い込まれてしまう。
修正：すべて && に直す。a-diff='A', z-diff='Z'。 
if(input >= a-diff && input <= z-diff)  printf("英字大文字です\n");
else if(input >= a && input <= z)        printf("英字小文字です\n");
else if(input >= '0' && input <= '9')    printf("数字です\n");
else printf("英数字以外です\n");
理解度:
S2_3バグ修正入れ子if
「2/5の倍数」を判定し、該当メッセージを「のみ」表示したい。次のコードは意図通りか？ 正しければ 4/7/10 の出力を、誤りなら箇所を指摘し修正コードを書け（論理演算子は使わない）。
if(x % 2 == 0){
    printf("2の倍数です\n");           // ← ここが常に出てしまう
    if( x % 5 == 0 ) printf("2と5の倍数です\n");
}
else{
    if( x % 5 == 0 ) printf("5の倍数です\n");
    else printf("2と5どちらの倍数でもないです\n");
}
✍️ 修正版を紙に書いてから開く

解答：誤り

x が2と5両方の倍数（例 10）のとき、「2の倍数です」と「2と5の倍数です」の2行が出てしまい「のみ」に反する。2の倍数です を else 側に移す。
if(x % 2 == 0){
    if( x % 5 == 0 ) printf("2と5の倍数です\n");
    else             printf("2の倍数です\n");
}
else{
    if( x % 5 == 0 ) printf("5の倍数です\n");
    else             printf("2と5どちらの倍数でもないです\n");
}

修正版での 4/7/10 の出力

4 → 2の倍数です 7 → 2と5どちらの倍数でもないです 10 → 2と5の倍数です
理解度:
S2_4空欄補充&& → 入れ子
if(x%4==0 && x>30) と同じ動作になるよう、入れ子版の①②を埋めよ。
if(   ①   ){
    if(   ②   ) printf("OK\n");
    else               printf("NG\n");
}
else printf("NG\n");

解答

① x % 4 == 0 ／ ② x > 30
A && B（両方真でOK）は「A を確かめ、真ならさらに B を確かめる」入れ子。① と ② は入れ替えても可。
理解度:
S2_5空欄補充|| → 入れ子
if(x%4==0 || x>30) と同じ動作になるよう、入れ子版の①②を埋めよ。
if(   ①   ) printf("OK\n");
else{
    if(   ②   ) printf("OK\n");
    else               printf("NG\n");
}

解答

① x % 4 == 0 ／ ② x > 30
A || B（どちらか真でOK）は「A が真なら即OK、偽なら else で B を確かめる」入れ子。① と ② は入れ替え可。
理解度:
S2_6空欄補充複合条件→入れ子★
if( (x%2==0 && x%4!=0) || (x%6!=0 && x%8==0) ) と同じ動作の入れ子版。①〜④を埋めよ。
if( ① ){
    if( ② ){ printf("OK\n"); }
    else{
        if( ③ ){
            if( ④ ){ printf("OK\n"); }
            else{ printf("NG\n"); }
        }
        else{ printf("NG\n"); }
    }
}
else{ printf("NG\n"); }

解答（全数一致を実行で確認済み）

① x % 2 == 0 ／ ② x % 4 != 0 ／ ③ x % 6 != 0 ／ ④ x % 8 == 0

なぜこの順？ 2つの AND 項はどちらも「x は偶数」を含む（x%8==0→x は偶数）。だから外枠①で x%2==0 を共通の門にできる。門を通ったら「x%4!=0 なら即OK（1つ目の項）、そうでなければ x%6!=0 かつ x%8==0（2つ目の項）」を入れ子で確かめる形。

理解度:
S2_7空欄補充うるう年★
うるう年判定：4で割り切れる→うるう年、ただし100で割り切れる→違う、ただし400で割り切れる→うるう年。入れ子版①〜③と、1文版④を埋めよ。
// sample7_1（入れ子）
if( ① )  printf("うるう年です\n");
else{
    if( ② )  printf("うるう年ではないです\n");
    else{
        if( ③ )  printf("うるう年です\n");
        else               printf("うるう年ではないです\n");
    }
}
// sample7_2（1文）
if( ④ )  printf("うるう年です\n");
else       printf("うるう年ではないです\n");

解答（1〜3000年で全数一致を確認済み）

① y % 400 == 0 ／ ② y % 100 == 0 ／ ③ y % 4 == 0
④ (y % 4 == 0 && y % 100 != 0) || y % 400 == 0

入れ子版は例外の強い順（400→100→4）に調べるのがコツ。400で割れれば無条件うるう年、次に100で割れれば例外で非うるう年、次に4で割れればうるう年、それ以外は非うるう年。

理解度:

4章 制御(2) ループ S3

for/while を1周ずつ手で追うのが本番。ループ回数、2重ループ、そして「for のヘッダ(初期化・条件・更新)を埋める」問題が主役。

🪙 ループの反射

  • 回数固定 for(i=0; i<n; i++) は n 回回る（i=0..n-1）。i=1; i<=n なら i=1..n の n 回。
  • 2重ループで重複しないペアは内側を for(j=i+1; j<=上限; j++)。
  • 「終わったとき i がいくつか」を聞く問題は、条件が偽になった瞬間の i を答える（そこで i++ 済みか要確認）。
S3_1空欄補充for・表出力
1〜10 の値とその2乗を表形式で出す。for のヘッダと printf の引数を埋めよ。
for( i = 0; ________ )
    printf(" %d   %d \n", ________ );
✍️ 紙に for 行と printf 行を書いてから開く

解答（どちらでも可）

// i=0 スタートを活かすなら
for( i = 0; i < 10; i++ )
    printf(" %d   %d \n", i+1, (i+1)*(i+1) );

// 素直に i=1 から回すなら
for( i = 1; i <= 10; i++ )
    printf(" %d   %d \n", i, i*i );
値は i、2乗は i*i。10個出したいので回数は10回。
理解度:
S3_2空欄補充追い越し・for条件
A君70m/分、B君100m/分。A君が min 分先に出発。B君が初めて完全に追い越すのは何分後か。差を初期化する行と for の条件を埋めよ（本体は distance -= (100-70);）。
scanf("%d",&min);
________                        // distance の初期化
for( i = 0; ________ ; i++ ){
    distance = distance - ( 100 - 70 );
}
printf("B君は %d 分後にはA君を追い越しています\n", i);

解答

distance = 70 * min;               // A君の先行距離
for( i = 0; distance >= 0; i++ ){
    distance = distance - ( 100 - 70 );
}
最初の差は「A君が min 分×70m/分 進んだぶん」＝70*min。B君は毎分 100-70=30m ずつ差を詰める。差が負（B君が前）になった瞬間ループを抜け、その時の i が答え。
検算：min=1 → 先行70m、70/30 で3分後に追い越し → 出力 3（実行確認済み）。
理解度:
S3_3出力予測while・for回数
出力（i と k）を求めよ。
int i=0,j,k; int a=1;
while( a < 10000 ){ a = a*10; i++; }
printf("i = %d\n", i);
k=0;
for( j=i; j<10; j++ ){ k++; }
printf("k = %d\n", k);

解答（実行で確認済み）

i = 4 k = 6
while：a=1→10(i=1)→100(i=2)→1000(i=3)→10000(i=4)。ここで a<10000 が偽になり終了。i=4。
for：j=4 から j<10 まで＝ j=4,5,6,7,8,9 の6回。k=6。
理解度:
S3_4空欄補充for本体・条件分岐
毎年、残高÷500 の利息が付く。ただし経過年数が3で割り切れる年は 残高÷100 に変わる。20年後まで残高を表示。for の本体を埋めよ。
zan = init;
for( i = 1; i < 20; i++){
    ________               // 利息を残高に加える
    printf("%d 年後の預金残高は %d 円です\n", i, zan );
}
✍️ if〜else で利息を分岐する数行を紙に

解答

if( i % 3 == 0 )  zan = zan + zan/100;   // 3の倍数年は 1/100
else              zan = zan + zan/500;   // それ以外は 1/500
zan += zan/100; のように複合代入でも可。ポイントは「年 i が3で割り切れるか」で利息の分母を切り替えること。整数割り算なので端数は切り捨て（＝商）。
理解度:
S3_5作成2重ループ・実数
1≦a≦5, 1≦b≦10 の全組合せで ax-b=0 の解 x=b/a を出力（表示例は 1.000000 形式）。2重 for で完成させよ。
✍️ 外ループ a、内ループ b、sol の計算を紙に

解答

for( a = 1; a <= 5; a++ ){
    for( b = 1; b <= 10; b++ ){
        sol = (double)b / a;          // 実数の割り算にする
        printf("%dx-%d=0 の解は %f\n", a, b, sol);
    }
}
sol は double。整数割り算にしないのが肝：b/a のままだと切り捨てられるので、(double)b/a か 1.0*b/a。表示は %f（既定で小数6桁）。
理解度:
S3_6作成2重ループ・組合せ
i < j を満たす (i,j) を (1,2)…(5,6) の順に全て出力（1≦i<j≦6）。2重 for で完成させよ。

解答

for( i = 1; i <= 5; i++ ){
    for( j = i+1; j <= 6; j++ ){
        printf("(%d, %d)\n", i, j);
    }
}
重複しないペアの定石：内側 j を i+1 から回すと自動的に i<j になり、(2,1) のような逆順や (1,1) が出ない。i は最大5（j=6 と組めるのは i=5 まで）。

5章 関数 S4

プロトタイプ宣言・引数・戻り値の3点セット。関数を「呼ぶ側」と「作る側」を両方書けること、配列を引数に取る関数を追えることが問われる。

🪙 関数の型（これを丸暗記）

  • 宣言（main の前）：int sum(int, int);（引数の型だけでOK、末尾 ;）
  • 定義：int sum(int a, int b){ return a+b; }（戻り値の型・returnを忘れない）
  • 呼び出し：ans = sum(x, y);
  • 配列を渡す：定義 int f(int a[])、呼び出し f(arr)（& も [] も付けない）
S4_1作成関数・戻り値

6章 配列 S5

1次元・2次元配列、添字の i-1 オフセット、#define マクロ定数、2次元のトレース。添字を1つずつ手で追うのが基本。

🪙 配列の反射

  • 宣言 int a[N]; の添字は 0 〜 N-1。「n番目」を a[n-1] で表すオフセットに注意。
  • scanf で配列要素に入れるのは scanf("%d", &a[i]);（& 必要）。
  • 2次元 a[i][j]：i が行、j が列。対角成分は a[0][0], a[1][1], …。
  • #define N 5 は「N を 5 に置き換える」だけ。サイズもループ上限も N に統一すれば1箇所で変えられる。
S5_1出力予測添字・範囲チェック
tray[]={15,200,18,55,30}。入力 i で在庫を表示。入力によって出力がどう変わるか。
if( ( i <= 0 ) || ( i > 5) )
    printf(" 1 から 5 までの数字を入力して下さい\n");
else
    printf(" %d 個の在庫があります\n", tray[i-1] );

解答

i が 1〜5 のとき tray[i-1] の在庫を表示：
1→15個, 2→200個, 3→18個, 4→55個, 5→30個。
i が 0以下 または 6以上のときは 1 から 5 までの数字を入力して下さい。

S5_3作成2次元配列・行列

char letter[6] = {'b','w','k','z','j','a'};
int i, j; char tmp;
for(i = 0; i < 6-1; i++){
    for(j = 0; j < 6-1-i; j++){
        if( letter[j] > letter[j+1] ){    // 隣と比べて大きければ交換
            tmp = letter[j];
            letter[j] = letter[j+1];
            letter[j+1] = tmp;
        }
    }
}
// → a b j k w z の順になる
